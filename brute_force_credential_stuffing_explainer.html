① Rate limiting — per account AND per IP Essential
Rate limit at multiple levels simultaneously. Per-IP alone is bypassed by IP rotation; per-account alone is bypassed by spraying. Both together make attacks significantly slower and more expensive.
## Express.js — layered rate limiting const rateLimit = require('express-rate-limit'); // Per-IP global limit app.use('/login', rateLimit({ windowMs: 60000, max: 20 })); // Per-account limit (track in Redis) async function checkAccountLimit(username) { const key = `login_attempts:${username}`; const attempts = await redis.incr(key); await redis.expire(key, 900); // 15 min window if (attempts > 10) throw new Error('Account temporarily locked'); } // Also: progressive delays (exponential backoff) const delay = Math.min(1000 * 2 ** attempts, 30000); await sleep(delay); // 1s, 2s, 4s, 8s... up to 30s
② Block X-Forwarded-For IP spoofing Essential
Never trust client-supplied IP headers for rate limiting. Only trust the real IP from the TCP connection, or from a trusted upstream proxy.
## Express.js — only trust proxy if you have one // BAD — trusts X-Forwarded-For from anyone: app.set('trust proxy', true); // GOOD — only trust proxies you control: app.set('trust proxy', '10.0.0.0/8'); // only internal proxies // Or get real IP directly (no trust proxy): const ip = req.socket.remoteAddress; ## Nginx — set real IP from your known upstream proxy: set_real_ip_from 10.0.0.1; # your load balancer IP only real_ip_header X-Forwarded-For; # All other X-Forwarded-For headers are ignored
③ Generic error messages + constant-time comparison Blocks enumeration
Always return the same message for invalid username and invalid password. Add a time delay to make timing-based enumeration impossible.

④ Compromised password detection Modern best practice
Check passwords against Have I Been Pwned at registration and login. NIST SP 800-63B recommends blocking passwords found in breach databases.
## Python — HIBP k-anonymity API check import hashlib, requests def is_pwned(password): sha1 = hashlib.sha1(password.encode()).hexdigest().upper() prefix, suffix = sha1[:5], sha1[5:] r = requests.get(f'https://api.pwnedpasswords.com/range/{prefix}') return any(line.split(':')[0] == suffix for line in r.text.splitlines()) # Enforce at registration: if is_pwned(new_password): raise ValueError("This password has appeared in a data breach. Choose a different one.")
⑤ Defence checklist
Rate limit per IP AND per account — not just one  ·  Never trust X-Forwarded-For from untrusted sources for rate limiting  ·  Generic error: "Invalid username or password" — never distinguish  ·  Constant-time auth to prevent timing-based enumeration  ·  Block breached passwords via HIBP API at registration  ·  Require MFA on all accounts — brute force becomes useless  ·  Log failed attempts with IP, username, timestamp — alert on spikes  ·  CAPTCHA after 3 failures — hard to remove entirely, harder to bypass at scale  ·  Alert users on login from new IP/device/country
